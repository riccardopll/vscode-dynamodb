# Agent Guidelines

- Always strive for concise, simple solutions that keep code understandable.
- If a problem can be solved in a simpler way, propose it.
- Avoid clever abstractions, over-optimization, and style hacks when a straightforward solution is clearer.
- Do not run `npm run build` after every small change. Only run it when the user explicitly asks for verification, when changes are broad enough to justify full validation, or when a build is needed to investigate/fix a problem.
